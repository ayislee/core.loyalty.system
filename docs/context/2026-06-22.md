# Context 2026-06-22

## Laporan Loyalty per Partner

Implementasi berdasarkan `core.loyalty.system/issue.md` telah diselesaikan pada `core.loyalty.system` dan `cms.loyalty.system` tanpa perubahan pada `backend-mediacartz`.

### Core Loyalty

1. Migration `1781020000000_partner_loyalty_report_schema.js` menambahkan:
   - `point_histories.source_type`;
   - index `partner_id + source_type + created_at`;
   - snapshot `member_vouchers.partner_id`;
   - snapshot `member_vouchers.voucher_name_snapshot`;
   - index `member_vouchers.partner_id + created_at`.
2. Data lama dibackfill secara konservatif menjadi `partner_award`, `partner_adjustment`, `voucher_hold`, `voucher_release`, `voucher_redeem_legacy`, atau `legacy`.
3. Writer point baru mengisi `source_type` secara eksplisit.
4. `VoucherSnapshot` sekarang menyimpan partner dan nama voucher saat voucher diterbitkan.
5. Konfirmasi token membuat `member_vouchers` dan mengubah confirmation menjadi `confirmed` dalam satu database transaction dengan row lock.
6. Service `PartnerLoyaltyReportService` menghitung summary dan detail per bulan dengan pagination serta filter member.
7. User admin dapat melihat semua/satu partner. User partner selalu dipaksa memakai partner dari `user_partners`.
8. Endpoint baru:

```text
GET /api/v1/admin/reports/point-awards
GET /api/v1/admin/reports/voucher-sales
```

Query utama: `partner_id`, `year`, `month`, `page`, `rows`, dan `member_keyword`.

### CMS Loyalty

1. Halaman baru:

```text
/report/point-awards
/report/voucher-sales
```

2. Sidebar memiliki group `LAPORAN` dengan menu Pemberian Point dan Penjualan Voucher.
3. Kedua halaman menyediakan filter partner untuk admin, bulan, tahun, pencarian konsumen, summary, tabel detail, pagination, loading, empty, dan error state.
4. User partner tidak melihat selector partner; pembatasan utama tetap dilakukan backend.

### Definisi Bisnis

1. Pemberian point hanya menghitung `source_type = partner_award` dengan point positif.
2. Release hold voucher tidak dihitung sebagai pemberian point.
3. Satu row `member_vouchers` dihitung sebagai satu voucher terjual.
4. Point penjualan voucher memakai `member_vouchers.redeemed_point`.
5. Total point penjualan hanya metrik laporan, bukan saldo partner.
6. Laporan dijamin pada level partner. Data per toko belum tersedia secara konsisten dan tidak boleh mengubah `backend-mediacartz` tanpa konfirmasi.

### Verifikasi

1. Migration berhasil dijalankan pada database development.
2. Smoke test read-only service bulan berjalan menghasilkan:
   - 8 transaksi pemberian point;
   - total 3.593 point diberikan;
   - 7 voucher terjual;
   - total 2.100 point terkumpul.
3. Route HTTP tanpa token menghasilkan `401`, membuktikan route terdaftar dan dilindungi JWT.
4. `npm run build` pada `cms.loyalty.system` berhasil.
5. Build hanya menampilkan warning lama tentang Browserslist dan dynamic import Dashboard.
6. `git diff --check` core dan CMS berhasil.
7. `backend-mediacartz` tidak berubah.
